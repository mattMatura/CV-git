Advanced Data & Architecture

The Modern Data Stack (ELT)

The shift from ETL (Extract-Transform-Load) to ELT (Extract-Load-Transform) decouples compute from storage (Snowflake/BigQuery). Transformations happen in-situ using SQL (dbt), treating data as code (version controlled assets).

Incremental Models

Recomputing full history is expensive. Use is_incremental() to process only new records based on timestamps.

Strategic Value: If dashboards are slow or expensive, the root cause is often a failure to implement efficient incremental logic.

Data Modeling Paradigms

  • Star Schema (Kimball): Fact Table surrounded by Dimension Tables. Optimized for BI (Tableau). Limit: Rigid; schema evolution is painful.
  • Activity Schema (Narrator): Single wide time-series table. Entity + Activity + Timestamp. Strategic Value: Enables foreign-keyless joins and "time-travel" queries for rapid product analytics.

Advanced SQL: The Language of Inquiry

Ranking
ROW_NUMBER(), RANK()

Assign sequential integers. Use to identify a user's "first" or "most recent" action.

Offset
LAG(), LEAD()

Access previous/next rows. Critical for calculating time intervals between events (e.g., signup -> purchase).

Modularity
WITH clause (CTEs)

Break monolithic queries into logical steps. Use Recursive CTEs for hierarchical data (org charts, referral trees).

RAG Architecture (Retrieval Augmented Generation)

The standard for grounding LLMs in proprietary data to prevent hallucinations.

1

Ingestion & Chunking

Split data (PDFs, Notion) into segments. Chunk size is a critical tuning parameter for context precision.

2

Vector Embeddings

Convert text to high-dimensional math vectors (e.g., OpenAI text-embedding-3). Semantic meaning = mathematical proximity.

3

Vector Database (ANN Search)

Store in Pinecone/Weaviate. Perform Approximate Nearest Neighbor search to retrieve relevant chunks.

4

Synthesis

Inject chunks into System Prompt. "Answer using only the provided context."

Orchestration & Configuration

  • LangChain: Modular abstractions for connecting LLMs to tools.
  • LangGraph: Graph-based architecture for stateful, cyclical flows (loops, retries).
  • Chain-of-Thought (CoT): Prompting model to "think step-by-step" to expose reasoning.
  • Few-Shot Prompting: Providing high-quality input-output examples to anchor behavior.

API Architectures: Strategic Comparison

Type Core Concept Strategic Strength Weakness
REST Resource-based (GET, POST). Returns fixed JSON. Universally supported, easy to cache, scalable. Over-fetching data.
GraphQL Query-based. Client asks for specific fields. Decouples frontend; eliminates over-fetching. Complex caching; steep learning curve.
gRPC Action-based. Binary serialization (Protobuf). Extremely low latency (internal microservices). Not human-readable; limited browser support.
Webhooks Event-driven. Server pushes to client URL. Real-time updates without polling. Client must run a server to receive; retry logic critical.
API-First Strategy: Define API contract (OpenAPI/Swagger) before writing code. Allows parallel frontend/backend dev and prevents "Backend-for-Frontend" silos.

Domain-Driven Design (DDD)

Ubiquitous Language ▼

Code language must match business language. If Business says "Customer" and Code says "User", bugs happen.

Bounded Contexts ▼

Defines logical boundaries. "Product" means something different to Inventory (weight) vs Marketing (copy). Separate them to prevent "God Objects". Align teams to these contexts (Conway's Law).

Event Storming:

Workshop to map "Domain Events" on a timeline. Best way to identify bottlenecks before coding.

SOLID Principles (Product Logic)

  • SRP: Single Responsibility. Avoid "kitchen sink" features.
  • OCP: Open/Closed. Add functionality by adding code, not rewriting old code.
  • LSP: Liskov Substitution. Consistent behavior for subclasses (e.g., all Notifications behave similarly).
  • ISP: Interface Segregation. Don't force mobile apps to consume desktop APIs.
  • DIP: Dependency Inversion. Business rules shouldn't depend on database details.

Cloud Native

Containerization (Docker) Packages code + dependencies. Solves "it works on my machine".
Orchestration (K8s) Manages the fleet (Pods, Services). Enables auto-scaling and self-healing. High complexity tax.
Serverless Pay-per-use (0 cost for idle). Good for bursty workloads. Risk: Cold starts.

FinOps

Shifting cloud spend from CapEx to OpEx. Accountability for variable spend.

  • Unit Economics: Optimize cost per transaction, not just total bill.
  • Tagging: Every dollar must be tagged to a team/product.
  • Forecasting: Predict variable spend to avoid "bill shock".

Security Protocols

OAuth 2.0 (Authorization) Access. "Valet Key". Allows app to act on user behalf.
OIDC (Authentication) Identity. "ID Badge". Login with Google.
Certifications:
• CISSP: Technical (Engineer)
• CISM: Governance/Risk (Executive)
II

Strategy & Economic Physics

Hamilton Helmer's 7 Powers

The statics of enduring differential returns. Without one, you are a commodity.

Scale Economies

Unit costs decline as volume increases. High fixed cost, zero marginal cost (Software).

Network Economies

Value increases as user base grows.
Distinction: Virality (Acquisition) vs Network Effect (Defensibility).

Counter-Positioning

New model incumbent cannot copy without damaging existing business (Netflix vs Blockbuster).

Switching Costs

Value loss if customer leaves. Operational entanglement, not just contracts.

Branding

Attribution of higher value to identical offering. Reduces risk (IBM effect).

Cornered Resource

Preferential access to coveted asset (Patents, Talent density, Data set).

Process Power

Embedded complex organization process that yields lower cost/superior product (Toyota).

Aggregation Theory

Power shifts to those who control demand. Zero marginal cost of distribution.

Level 1: Supply Acquisition Cost

Netflix. Must pay to produce/license supply. Scaling constrained by capital.

Level 2: Supply Transaction Cost

Uber. Does not own cars, but pays per ride. Scaling constrained by physical supply.

Level 3: Zero Supply Cost

Google/Facebook. Users provide supply for free. Pure aggregation. Max margins.

Network Effects (NFX 16 Types)

Not all moats are equal.

  • Direct: Metcalfe's Law. Telephones. Strongest.
  • 2-Sided: Marketplaces (Supply/Demand). Hard to crack (Chicken/Egg).
  • Data: Product gets smarter with data (Waze). Warning: Asymptotic (eventually data is "good enough" and moat weakens).
  • Tech Performance: Tech gets faster with more nodes (BitTorrent).

Positioning & GTM Architecture

Dunford's Positioning (10 Steps)

1. Ignore "Average" users; focus on Best-Fit.

2. List Competitive Alternatives (e.g., Excel, "Do nothing").

3. Isolate Unique Attributes.

4. Map Attributes to Value Clusters.

5. Determine Who Cares (ICP).

6. Find Market Frame of Reference (Context).

Strategic Narrative: Don't use "Positioning Statements". Use a narrative that sells a "Change in the World".

Modern GTM Motions

Product-Led Sales (PLS) Hybrid. Product hooks user (PLG) -> Data identifies PQL (Product Qualified Lead) -> Sales closes Enterprise deal.
Ecosystem-Led Growth (ELG) Map customer overlaps with partners (Crossbeam). Borrow trust. Embed in solution mesh.
API-First GTM DevEx is marketing. Minimize "Time to First Call". Bottom-up developer adoption -> Top-down enterprise control.

Financial Engineering (Metrics 2.0)

Rule of 40

Growth % + Profit % >= 40

Weighted Rule: (1.33 x Growth) + (0.67 x Profit). Favors growth but demands efficiency. Trade-off: 100% growth can burn 60%. 20% growth must yield 20% profit.

Burn Multiple

Net Burn / Net New ARR

  • < 1.0x: Efficient (Top Decile)
  • 1.5x - 2.0x: Suspect
  • > 3.0x: Crisis

Retention Dynamics

GRR (Gross)

Excludes upsells. The "Truth Teller" of Product-Market Fit. Target 90%+.

NRR (Net)

Includes upsells. The "Growth Engine". Target 120%+. Warning: High NRR can mask poor GRR.

Pricing & Capital Structure

Usage-Based Pricing

Aligns price with value. Uncapped upside.

Game Theory:

Primary unit. Aligned to user journey. "You build it, you run it."

3. Enabling Teams ▼
Internal consultants (e.g., Security, Architecture) who upskill streams.

Executive Tactics

Negotiation (The Black Swan)
  • Mirroring: Repeat last 3 words. Builds connection.
  • Labeling: "It seems like..." Validates emotion.
  • Accusation Audit: List all negatives upfront. Disarms objections.
  • No > Yes: "Have you given up on...?" makes people feel safe.
Decision Velocity
  • Amazon 6-Pager: Silent start meetings. Narrative forces clarity.
  • NCTs vs OKRs: Narratives, Commitments (100% binary), Tasks. Connects strategy to tickets better than OKRs.
